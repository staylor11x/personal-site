# DESIGN_CONSTITUTION.md

## Core Identity

This website is a retro-futurist personal operating system.

It should feel:
- technical
- cinematic
- atmospheric
- industrial
- modern
- precise

The aesthetic direction is:
- synthwave
- retro-futurism
- cyberpunk
- geometric modernism
- industrial sci-fi
- 1980s future concept art

The website should feel like:
> a futuristic engineer's personal terminal node

NOT:
- a startup landing page
- a generic portfolio template
- a neon gimmick site
- an over-animated hacker parody

---

# Visual Philosophy

## Primary Principles

Prioritize:
1. atmosphere
2. clarity
3. geometry
4. intentionality
5. performance
6. consistency

Every visual element should feel:
- engineered
- deliberate
- sharp
- premium

Avoid playful or soft styling.

---

# Shape Language

## Geometry Rules

Use:
- sharp edges
- clipped corners
- beveled panels
- angular layouts
- segmented containers
- asymmetry where appropriate

Inspiration:
- DeLorean body panels
- aircraft instrumentation
- sci-fi control panels
- industrial hardware
- retro arcade systems

Avoid:
- oversized rounded corners
- circular blob layouts
- soft cards
- floating pill buttons
- overly organic shapes

---

# Color System

## Primary Colors

Backgrounds:
- deep navy
- near-black blue
- dark graphite

Accent Colors:
- vivid magenta
- neon pink
- electric cyan
- vivid green
- royal blue

Text:
- off-white
- muted gray
- cool silver

---

## Color Rules

Accent colors are used sparingly.

Neon should:
- guide attention
- highlight interactions
- create edge lighting
- enhance atmosphere

Do NOT:
- saturate the full UI
- use rainbow gradients
- use pastel palettes
- use washed-out blue-only themes

Dark surfaces should dominate.

---

# Lighting

Use:
- edge glow
- subtle emissive lighting
- directional highlights
- neon reflections
- monitor-style illumination

Avoid:
- excessive blur
- foggy gradients
- soft white shadows
- glassmorphism overload

Glow should feel:
- electrical
- controlled
- cinematic

---

# Typography

## Primary Fonts
- Inter
- Geist

## Monospace Fonts
- JetBrains Mono

## Typography Rules

Use monospace for:
- terminals
- labels
- metadata
- UI accents
- system messaging

Use sans-serif for:
- readable content
- long-form text
- navigation

Typography should feel:
- clean
- technical
- minimal

Avoid:
- playful typography
- handwritten fonts
- excessive font variation

---

# Motion Design

## Animation Philosophy

Motion should feel:
- mechanical
- intentional
- smooth
- system-driven

Use:
- sliding panels
- scan animations
- tactical reveals
- subtle hover transitions
- directional movement

Avoid:
- bounce animations
- floaty motion
- elastic easing
- exaggerated scaling
- playful interactions

Animations must support the atmosphere, not dominate it.

---

# Layout Rules

## Spacing

Layouts should:
- breathe
- use strong alignment
- feel architectural
- maintain visual rhythm

Prefer:
- grids
- segmented sections
- layered panels
- structured spacing

Avoid:
- cramped layouts
- oversized padding everywhere
- excessive empty blur zones

---

# UI Components

## Cards

Cards should feel like:
- interface modules
- control panels
- system nodes

Preferred styling:
- angular edges
- subtle borders
- segmented surfaces
- thin neon highlights

Avoid:
- giant rounded rectangles
- generic Tailwind dashboard cards

---

## Buttons

Buttons should feel:
- tactile
- industrial
- precise

Use:
- sharp geometry
- thin borders
- subtle glow states
- strong hover contrast

Avoid:
- bubble buttons
- oversized CTA styles
- soft gradients

---

# Backgrounds

Backgrounds should be:
- dark
- textured subtly
- atmospheric

Allowed:
- grid overlays
- scanlines
- stars
- noise textures
- tactical patterns
- subtle gradients

Avoid:
- flat white sections
- bright backgrounds
- heavy blur meshes
- generic SaaS gradients

---

# Terminal Design

The terminal is a core identity element.

It should feel:
- authentic
- fast
- minimal
- futuristic

Use:
- monospace typography
- clean prompts
- subtle glow
- restrained animation

Avoid:
- fake hacker spam
- excessive matrix effects
- green-only terminal clichés

---

# Travel Globe

The globe should feel:
- cinematic
- ambient
- exploratory

Use:
- glowing arcs
- dark earth tones
- subtle rotation
- atmospheric lighting

Avoid:
- corporate dashboard visuals
- cluttered map interfaces

---

# Spotify Integration

Music UI should feel:
- integrated
- atmospheric
- secondary to the main layout

Use:
- waveform visuals
- subtle equalizers
- clean album presentation

Avoid:
- oversized music widgets
- bright Spotify green dominance

---

# Accessibility

Despite the aesthetic:
- readability is mandatory
- contrast must remain strong
- motion must be reducible
- mobile responsiveness is required

Never sacrifice usability for style.

---

# Performance Rules

The website must remain:
- lightweight
- responsive
- smooth

Avoid:
- unnecessary animation libraries
- unoptimized shaders
- excessive particle systems
- heavy autoplay effects

Performance is part of the aesthetic.

---

# Forbidden Patterns

Never introduce:
- pastel-heavy UI
- oversized rounded cards
- startup SaaS aesthetics
- excessive blur
- emoji-heavy interfaces
- random neon everywhere
- generic cyberpunk clichés
- cluttered dashboards
- glassmorphism overload
- inconsistent spacing
- multiple competing visual styles

---

# Emotional Goal

Visitors should feel:
- curiosity
- atmosphere
- technical confidence
- creativity
- individuality

The website should feel like:
> a premium futuristic engineering workspace

NOT:
> a trendy frontend experiment

---

# Final Rule

Every new feature, component, animation, or section must answer:

1. Does this strengthen the retro-futurist identity?
2. Does this feel engineered and intentional?
3. Is this visually consistent with the system?
4. Is the effect subtle enough to remain professional?
5. Does this improve atmosphere without harming usability?

If the answer is "no", redesign it.
